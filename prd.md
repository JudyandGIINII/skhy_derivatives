## 1. 요약

SK하이닉스 보통주(KRX/NXT `000660`), 미국 예탁주식(Nasdaq `SKHY`), 국내외 단일종목 레버리지 상품은 서로 다른 거래시간, 통화, 투자자 기반, 유동성 및 상품 운용 규칙을 가진다. 이 차이는 일시적인 가격 괴리와 예측 가능한 수급을 만들 수 있지만, 동시에 오래된 가격 비교, 불완전한 전환 경로, 합성 스왑의 헤지 방식, 차입 불가 및 높은 거래비용 때문에 겉보기 차익이 실제 수익으로 이어지지 않을 수 있다.

본 제품은 다음 세 가설을 서로 독립적으로 검증한다.

1. 레버리지 상품의 일일 목표배율 재설정이 KRX 종가 부근의 방향성 수급을 예측하는가.
2. `SKHY`와 `000660`의 환율·예탁비율 조정 가격 괴리가 전환 가능 상태에서 비용을 초과해 수렴하는가.
3. Nasdaq 프리마켓과 NXT 애프터마켓이 동시에 열리는 구간에 한 시장의 가격발견이 다른 시장보다 유의하게 빠른가.

제품은 가설별 데이터 계보, 신호, 모의 주문, 모의 체결, 비용, 리스크 판정 및 실험 결과를 보존한다. 사용자는 각 전략을 `통과`, `보류`, `폐기` 중 하나로 판정할 수 있어야 한다.

## 2. 문제 정의

### 2.1 현재 문제

- 초안의 일부 수치는 공식 원자료가 없고, 가격 괴리를 계산할 때 서로 다른 시점의 종가가 혼용되어 있다.
- 비싼 ADR의 전통적 수렴 거래 방향이 반대로 서술되어 있다. 경제적 등가비율은 보통주 1주 대 ADR 10주이므로, ADR 프리미엄 수렴의 기본 방향은 `000660 매수 + SKHY 매도/헤지`다.
- 홍콩 `7709`는 본주를 단순히 2배 보유하는 현물 ETF가 아니라 스왑 기반 합성 상품이다. 공개 AUM 전체가 같은 시각에 본주 주문으로 전환된다고 볼 수 없다.
- `SKHY` 상장 이후의 관측 기간이 매우 짧다. 짧은 구간의 높은 수익률을 CAGR로 연율화하면 전략 성능을 과장한다.
- 기관 수준의 전환·대차·저지연 데이터 접근성을 개인 투자자에게 그대로 가정할 수 없다.

### 2.2 해결하려는 문제

공식 및 사용 허가된 데이터 소스를 통합하여 다음 질문에 비용 후 기준으로 답한다.

- 신호 생성 시점에 실제로 알 수 있었던 정보만으로 결과가 재현되는가.
- 이론적 수급과 실제 종가 체결 사이에 안정적인 관계가 있는가.
- 괴리가 단순한 시차·환율·스프레드인지, 거래 가능한 상대가치 기회인지 구분할 수 있는가.
- 차입비용, 세금, 수수료, 스프레드, 슬리피지 및 한쪽 다리 미체결을 반영한 뒤에도 기대값이 양수인가.
- 데이터나 실행 조건이 나쁠 때 거래하지 않는 판단을 자동으로 내릴 수 있는가.

## 5. 검증된 시장 구조와 사실 경계

### 5.1 ADR 등가비율과 가격

SEC 공모 문서상 `SKHY` ADR 1주는 SK하이닉스 보통주 0.1주를 표창한다. 따라서 ADR 10주와 보통주 1주가 경제적으로 대응한다.

환율 `USD/KRW`를 1달러당 원화 수로 정의할 때:

```text
adr_fair_usd(t) = kr_common_krw(t) / usdkrw(t) / 10
premium(t) = skhy_usd(t) / adr_fair_usd(t) - 1
```

표시용 중간값과 거래 가능 괴리를 구분한다.

```text
executable_entry_premium
  = (skhy_bid_usd * 10 * usdkrw_bid - kr_ask_krw) / kr_ask_krw
    - estimated_entry_cost_rate

executable_exit_premium
  = (skhy_ask_usd * 10 * usdkrw_ask - kr_bid_krw) / kr_bid_krw
    + estimated_exit_cost_rate
```

- `t`는 동기화 허용범위 안의 시점이어야 한다.
- 한 시장이 닫혀 있으면 마지막 종가 기반 괴리를 `stale_reference`로 표시하며 거래 가능 괴리로 부르지 않는다.
- ADR 프리미엄이 양수일 때 기본 상대가치 방향은 보통주 롱과 ADR 숏이다.
- 공매도·대주가 불가능하면 이를 무위험 차익으로 간주하지 않고 신규 페어 진입을 차단한다.

### 5.2 2026년 7월 29일의 의미

공식 KRX 공시는 2026년 7월 29일을 ADR 원주 신주의 KRX 추가상장 예정일로 기재한다. 이는 개인·기관의 양방향 전환 서비스가 반드시 같은 날 운영된다는 보장이 아니다.

전환 상태는 다음 열거형으로 관리한다.

| 값 | 의미 | H2 신규 진입 |
| --- | --- | --- |
| `UNKNOWN` | 공식 운영 상태를 확인하지 못함 | 금지 |
| `ANNOUNCED` | 회사·예탁기관이 예정만 공지 | 금지 |
| `OPERATIONAL` | 예탁기관과 실제 사용할 브로커가 접수 가능, 비용·처리기간을 확인 | 허용 가능 |
| `SUSPENDED` | 접수 중단·한도 소진·법규상 제한 | 금지 |

`OPERATIONAL` 판정에는 Citi/KSD 공지와 실제 사용 브로커의 접수 가능 여부, 최소수량, 수수료, 예상 결제기간이 모두 필요하다. 언론 또는 SNS 단독 보도는 상태를 변경하지 못한다.

### 5.3 레버리지 상품

- HKEX `7709`는 스왑 기반으로 SK하이닉스 보통주의 일일 수익률 2배를 목표로 하며 KRX 종가 부근에서 일일 재조정을 수행한다.
- 실제 본주 헤지는 스왑 상대방의 재량·기존 포지션·대체 헤지·내부 상계의 영향을 받는다.
- 국내 단일종목 레버리지 ETF·ETN도 상품별 설정 방식, 현물·선물·스왑 구성과 설정·환매가 다르므로 상품별 계약과 PCF를 저장한다.
- AUM은 잠재 수급 규모의 상한 또는 설명변수이지 확정 주문량이 아니다.

### 5.4 배제하거나 조건부로만 쓰는 주장

다음 수치는 공식 원자료가 확인될 때까지 사실 테이블과 신호에서 배제한다.

- 120만 계좌 마진콜, 32만~36만 계좌 강제청산
- 청산 계좌의 62%가 20~30대라는 주장
- 투자자예탁금이 한 달 만에 140조원에서 107조원으로 감소했다는 주장
- 레버리지 청산이 기관 순매도의 62%를 차지했다는 주장
- 특정 날짜 이후 개인도 ADR 전환을 무조건 신청할 수 있다는 주장

이 정보가 UI에 노출될 경우 `UNVERIFIED_SOCIAL_CLAIM`로 표시하고 출처 URL·게시시각·검증 상태를 함께 보여준다.

### 7.2 공급자 충돌 규칙

- 종목코드, 상장상태, 기업행사와 공식 종가는 KRX·발행사 자료를 우선한다.
- 실시간 페이퍼 체결은 실제 선택한 거래소·세션의 KIS 호가를 기본으로 하고 토스 호가로 이상 여부를 대조한다.
- 서로 다른 세션 또는 조정방식의 가격은 직접 비교하지 않는다.
- 공급자 간 차이가 허용범위를 넘으면 `SOURCE_DIVERGENCE`를 기록하고 신규 신호를 막는다.
- 네이버·Yahoo의 지연 데이터로 실시간 공급자를 자동 대체하지 않는다.
- 각 공급자의 호출 제한·재배포·저장 조건을 준수하며 원문 이용약관 URL과 확인일을 데이터 카탈로그에 저장한다.

### 7.3 인증정보

- API 키, 시크릿, 접근토큰, 계좌번호를 Git 추적 파일에 저장하지 않는다.
- 환경변수 또는 운영체제 시크릿 저장소에서 주입한다.
- 로그, 예외 메시지, 리포트, 테스트 픽스처에서 민감값을 마스킹한다.
- v1은 가능한 경우 조회 전용 키를 사용한다.
- 토스·KIS 계좌·주문 엔드포인트는 코드 수준에서 비활성화하고 `BrokerProvider` 실주문 구현을 등록하지 않는다.

### 7.4 공급자 인터페이스

수집·기준정보·백필·체결 책임을 다음 포트로 분리한다. 공급자 SDK의 응답 객체를 전략 코드에 직접 노출하지 않는다.

| 인터페이스 | 최소 책임 | v1 구현 규칙 |
| --- | --- | --- |
| `MarketDataProvider` | 실시간 quote/trade/예상체결 구독, 구독 해제, 연결상태·지연 보고, 재연결 | KIS 기본, Toss 교차검증; 지연 포털 데이터로 자동 대체 금지 |
| `ReferenceDataProvider` | 종목 마스터, 거래소·세션, 기업행사, 상품 구조, ADR 비율·전환 상태 조회 | KRX·SEC·KIND·HKEX·발행사 원문 우선 |
| `HistoricalDataProvider` | 기간·주기·조정방식이 명시된 bar/통계 백필과 데이터 계보 반환 | KRX 기준 원장, Naver/Yahoo는 보조·대조용 |
| `BrokerProvider` | 계좌·주문·취소·체결 이벤트의 추상 계약 | v1에는 `PaperBrokerProvider`만 등록; KIS/Toss 실주문 어댑터는 빌드·주입하지 않음 |

각 구현은 capability, 호출 제한, 지연 특성, 라이선스·저장 가능 범위와 마지막 확인일을 노출해야 한다. 지원하지 않는 기능은 빈 데이터로 위장하지 않고 명시적 `UNSUPPORTED_CAPABILITY` 오류를 반환한다.

## 8. 공통 데이터 계약

### 8.1 공통 규칙

- canonical timestamp는 UTC epoch nanoseconds(`int64`)이며 화면에는 ISO 8601을 병기한다.
- 원시장 시간대와 원본 timestamp를 별도 보존한다.
- 금액·가격·수량 계산에는 이진 부동소수점 대신 고정소수점 또는 `Decimal`을 사용한다.
- 통화는 ISO 4217(`KRW`, `USD`, `HKD`)을 사용한다.
- 원천 데이터는 불변 저장하고 정규화·보정 결과는 별도 계층에 쓴다.
- 모든 파생값은 입력 레코드 ID, 알고리즘 버전과 설정 해시를 가진다.

모든 시장·기준·과거 데이터 레코드는 다음 `RecordEnvelope`를 포함한다. 값이 적용되지 않는 경우에도 필드를 생략하지 않고 `null`과 사유코드를 저장한다. `event_time_utc`와 `received_time_utc`의 직렬화 타입은 UTC epoch nanoseconds `int64`다.

| 필드 | 의미 |
| --- | --- |
| `source` | 원천 공급자·공식기관 |
| `venue` | `KRX/NXT/NASDAQ/HKEX/OTC/REFERENCE` 등 |
| `symbol` | 원천 심볼; 내부 `instrument_id`와 함께 저장 |
| `event_time_utc` | 거래소 이벤트 또는 기준정보 효력 시각 |
| `received_time_utc` | 시스템 수신 시각 |
| `currency` | ISO 4217 통화; 비금전 레코드는 `null`과 사유코드 |
| `session` | `PRE/REGULAR/CLOSE_AUCTION/AFTER/CLOSED/REFERENCE` |
| `is_delayed` | 공급자 정의 및 측정 지연 여부 |
| `adjustment_status` | `RAW/ADJUSTED/NOT_APPLICABLE/UNKNOWN` |
| `quality_flag` | 0개 이상의 품질 플래그 배열 |

### 8.2 핵심 타입

#### `MarketQuote`

| 필드 | 단위/값 |
| --- | --- |
| `instrument_id` | 내부 안정 식별자 |
| `symbol`, `venue` | 원시장 심볼, `KRX/NXT/NASDAQ/HKEX` |
| `session` | `PRE/REGULAR/CLOSE_AUCTION/AFTER/CLOSED` |
| `bid_price`, `ask_price` | 해당 통화 가격 |
| `bid_size`, `ask_size` | 주·좌·계약 수량 |
| `event_time_utc` | 거래소 이벤트 시각, UTC epoch nanoseconds |
| `received_time_utc` | 로컬 수신 시각, UTC epoch nanoseconds |
| `source` | `KIS/TOSS/KRX/NAVER/YAHOO/ISSUER` 등 |
| `currency` | `KRW/USD/HKD` 등 |
| `is_delayed` | 지연 여부 |
| `adjustment_status` | quote는 일반적으로 `RAW` |
| `quality_flag` | 0개 이상의 품질 플래그 배열 |

#### `Trade`와 `Bar`

- `Trade`는 가격, 수량, 매수·매도 방향(제공될 때만), 거래소·세션, 이벤트·수신시각을 가진다.
- `Bar`는 `1m/5m/1d` 등 명시적 주기, OHLCV, 거래대금, 원시·수정 여부와 생성 기준을 가진다.
- 서로 다른 공급자의 bar를 조용히 이어 붙이지 않고 구간별 출처를 기록한다.

#### `FundSnapshot`

| 필드 | 의미 |
| --- | --- |
| `fund_id` | 레버리지 상품 내부 ID |
| `leverage_beta` | 목표 일일 배율 |
| `aum`, `nav`, `inav` | 통화와 기준시각을 포함한 값 |
| `shares_outstanding` | 발행좌수, 제공될 때 |
| `net_creation_estimate` | 설정·환매 추정치와 추정방법 |
| `replication_type` | `PHYSICAL/FUTURES/SWAP/MIXED/UNKNOWN` |
| `published_at` | 실제 공개시각 |
| `effective_at` | 값의 기준시각 |

#### `FXQuote`

- 통화쌍 방향은 항상 `USD/KRW = 1 USD당 KRW`로 정의한다.
- bid/ask와 이벤트시각을 저장한다.
- 일별 매매기준율은 실시간 실행 환율과 구분한다.

#### `ConversionStatus`와 `BorrowQuote`

- `ConversionStatus`는 상태, ADR 비율, 최소수량, 수수료, 예상 처리기간, 공식 근거 URL, 확인시각을 가진다.
- `BorrowQuote`는 차입 가능수량, 연율 비용, 리콜 조건, 공급자와 유효시각을 가진다.
- 두 값이 없거나 만료되면 H2의 페어를 거래 가능 상태로 판정하지 않는다.

#### `Signal`, `OrderIntent`, `PaperFill`

- `Signal`: 전략 ID·버전, 방향, 신뢰도, 예상 총수익·비용·순수익, 만료시각, 입력 데이터 ID를 저장한다.
- `OrderIntent`: 페이퍼 전용이며 주문 다리, 지정가, 수량, 유효기간, 헤지비율과 리스크 예산을 가진다.
- `PaperFill`: 사용한 호가·거래량, 체결·미체결 수량, 슬리피지와 체결모델 버전을 저장한다.

#### `RiskDecision`과 `ExperimentResult`

- `RiskDecision`은 `ALLOW/BLOCK/REDUCE`와 사유코드, 적용 전후 수량, 한도 상태를 기록한다.
- `ExperimentResult`는 데이터 버전, 분할 구간, 비용 가정, 지표, 신뢰구간, 스트레스 결과와 `PASS/HOLD/REJECT`를 가진다.

### 8.3 품질 플래그

최소 다음 값을 지원한다.

`STALE`, `DELAYED`, `OUT_OF_ORDER`, `DUPLICATE`, `GAP`, `SOURCE_DIVERGENCE`, `MARKET_CLOSED`, `HALTED`, `UNADJUSTED_CORPORATE_ACTION`, `UNKNOWN_CONVERSION`, `BORROW_UNAVAILABLE`, `UNVERIFIED_SOCIAL_CLAIM`.

## 9. 전략 가설 요구사항

### 9.1 H1 — 종가 리밸런싱 충격

#### 가설

일일 목표배율 `β`를 추종하는 상품의 장중 수익률 변화는 종가 부근 목표 노출 조정을 만든다. 순설정·환매와 비용을 무시한 1차 근사는 다음과 같다.

```text
theoretical_delta_exposure_i(t)
  ≈ beta_i * (beta_i - 1) * prior_nav_i * underlying_return_i(t)
```

실제 사용 모델은 설정·환매, 상품 구조, 이미 수행된 헤지와 스왑 상대방의 상계를 반영한다.

```text
estimated_close_pressure(t)
  = sum(kappa_i,regime * theoretical_delta_exposure_i(t)
        + observable_flow_adjustment_i(t))
    / underlying_20d_adv_notional
```

`kappa`는 상품·국면별 실제 전이계수이며 학습 구간에서만 추정한다. 합성상품에 `kappa=1`을 가정하지 않는다.

`observable_flow_adjustment`는 해당 시점까지 관측 가능한 종가 예상체결 불균형, 프로그램매매 순매수, 설정·환매 추정과 상품별 복제방식으로 구성한다. 사후 검증에서는 실제 종가 경매 거래대금·불균형 대용치, 15:20 이후 잔여수익과 `000660` ADV 대비 압력을 각각 비교한다. 특정 경매·프로그램 피드를 확보하지 못하면 0으로 대체하지 않고 결측 플래그와 별도 모델 버전을 사용한다.

#### 의사결정 시점과 입력

- 기본 신호 스냅샷: KRX 15:10:00 KST.
- 신규 페이퍼 주문 의도 생성 마감: 15:19:30 KST.
- 입력: 그 시점까지의 `000660` 수익률·거래량·호가, 전일까지 공개된 AUM/NAV, 당일까지 공개된 설정·환매, 상품별 배율·복제방식, 프로그램매매, 종가 예상체결 정보.
- 당일 장 종료 후 확정되는 NAV·AUM을 같은 날 신호에 사용하지 않는다.

#### 페이퍼 실행

- 15:10~15:20 사이 지정가 진입, KRX 종가 경매 또는 사전 정의한 NXT 청산 규칙으로 종료한다.
- 매수·매도 양방향을 별도 성과로 보고한다.
- 가격제한폭 확대, 변동성완화장치, 거래정지, 예상체결 피드 부재 시 차단한다.

#### 반증 기준

- 학습 외 구간에서 압력 방향과 종가 잔여수익의 관계가 비용 후 사라진다.
- 결과가 소수 1~3일 또는 한 상품의 AUM 수정치에 집중된다.
- 2배 비용 스트레스에서 누적손익이 음수다.
- 이론 흐름 대비 실제 종가 거래의 부호 일치가 무작위 수준을 벗어나지 못한다.

### 9.2 H2 — ADR 프리미엄 수렴

#### 가설

동시 거래가 가능하고 보통주→ADR 전환이 실제로 운영될 때, 비용을 크게 초과하는 ADR 프리미엄은 보통주 롱·ADR 숏 수요로 축소될 수 있다.

#### 진입 필수조건

- NXT와 Nasdaq 프리마켓이 모두 열려 있고 양쪽 호가가 신선하다.
- `ConversionStatus=OPERATIONAL`이다.
- 유효한 `BorrowQuote`가 있거나 사전 승인된 대체 헤지의 실시간 호가가 있다.
- 1 보통주 대 10 ADR 경제적 비율을 기준으로 양쪽 최소 거래단위를 충족한다.
- 스프레드, FX, 수수료, 세금, 차입비용, ADR 발행·취소비용과 예상 슬리피지를 차감한 순괴리가 양수다.
- 양쪽 다리를 동일한 원자적 주문처럼 시뮬레이션하며 한쪽만 체결될 위험 예산을 통과한다.

#### 종료조건

- 실행 가능 프리미엄이 비용 버퍼 이하로 수렴한다.
- 전환·차입 상태가 악화되거나 리콜 위험이 발생한다.
- 최대 보유기간 또는 손실 한도에 도달한다.
- 한쪽 시장의 거래정지·가격제한·데이터 장애가 발생한다.

#### 대체 헤지

- `SKHY` 직접 숏이 불가능하면 `SKHZ` 등 인버스 상품이나 정의된 손실의 옵션 조합을 별도 전략 변형으로만 평가한다.
- 일일 인버스 상품의 복리·추적오차·스왑 비용을 직접 숏과 동일하게 취급하지 않는다.
- 헤지 수단이 없으면 보통주 단독 롱을 H2 차익거래로 실행하지 않는다.

#### 반증 기준

- 전환이 운영되어도 프리미엄이 비용 후 안정적으로 수렴하지 않는다.
- 차입·FX·스프레드 비용이 예상 총수익의 대부분을 소모한다.
- 한쪽 다리 미체결과 시장 비동시성이 손익을 지배한다.
- 소수의 ADR 상장 초기일을 제외하면 신호가 재현되지 않는다.

### 9.3 H3 — NXT–Nasdaq 동시 가격발견

#### 가설

미국 서머타임 기준 Nasdaq 프리마켓 개장 이후 NXT 애프터마켓 종료 전까지 두 시장이 겹친다. 이 구간에 `SKHY` 또는 `000660` 중 한쪽이 공통 정보를 먼저 반영하고 상대 시장이 지연 반응할 수 있다.

#### 요구사항

- 거래시간을 고정 KST 문자열로 저장하지 않고 거래소 캘린더와 DST로 계산한다.
- 1초·5초·1분 동기화 표본을 별도로 유지한다.
- 환율, Nasdaq 반도체 지수, Micron 등 공통 메모리 요인을 통제한 잔여수익을 계산한다.
- 양방향 Granger/lead-lag, 교차상관, 이벤트 응답과 거래 가능한 모의손익을 함께 측정한다.
- 연구 결과가 선행성을 보이더라도, 신호 시점 이후의 실행 호가로만 체결한다.
- 최대 30분 또는 동시거래 종료 전까지의 단기 보유를 기본으로 하고 장외 보유는 별도 변형으로 분리한다.

#### 반증 기준

- 선행성은 존재하지만 첫 실행 가능 호가에서 이미 사라진다.
- 저유동성 프리마켓의 비정상 체결 한두 건이 결과를 지배한다.
- 공통 반도체·환율 요인을 제거하면 예측력이 사라진다.
- 신호 빈도와 시장 깊이가 개인의 최소 주문에도 부족하다.

## 10. 백테스트 및 통계 검증

### 10.1 엔진 요구사항

- 이벤트 기반으로 복수 거래소·세션·통화를 처리한다.
- 틱, 호가, 분봉과 일봉의 혼합 이벤트를 지원한다.
- 부분체결, 주문 만료, 거래정지, 가격제한폭, 수수료, 세금, FX, 차입비용과 리콜을 모델링한다.
- 같은 timestamp 이벤트의 정렬 규칙을 명시하고 재실행 시 동일 결과를 낸다.
- 특정 라이브러리를 강제하지 않으며 구현 계획에서 적합한 엔진을 선정한다.

### 10.2 데이터 구간

- H1: 신뢰 가능한 관련 상품 데이터가 존재하는 전체 구간, 최소 120 KRX 거래일.
- H1 국면: 홍콩 상품만 존재하던 기간, 국내 단일종목 상품 상장 후, ADR 상장 후, 규제 보완 발표 후를 분리한다.
- H2/H3: `SKHY` 거래 개시일인 2026-07-10 이후만 실제 데이터로 사용한다.
- H2/H3의 짧은 역사는 과거 유사종목으로 합성해 SK하이닉스 실적처럼 보고하지 않는다. 유사사례는 방법 검증용 보조 실험으로만 표시한다.

### 10.3 분할과 튜닝

- 시간순 분할만 사용한다.
- H1 최소 120일은 최초 60일 학습, 다음 30일 검증, 마지막 30일 테스트를 기본으로 하며 이후 확장형 walk-forward를 추가한다.
- 임계값·전이계수·보유시간은 학습·검증에서만 선택하고 테스트 구간을 본 후 변경하지 않는다.
- 변경할 경우 전략 버전을 올리고 새로운 미사용 테스트 구간을 확보한다.
- 모든 실험은 데이터 스냅샷 ID, 코드 커밋, 설정 해시와 난수 시드를 저장한다.

### 10.4 비용 모델

최소 다음 비용을 포함한다.

- 국내·미국 매매수수료 및 적용 세금
- KRX/NXT/Nasdaq bid-ask spread
- 주문 크기와 호가 깊이에 따른 슬리피지·시장충격
- USD/KRW 환전 스프레드 및 환전수수료
- ADR 발행·취소·예탁 관련 비용
- 주식 대차비용, 변동 차입금리와 리콜
- 레버리지·인버스 상품의 추적오차와 보수

기본 비용과 각 비용을 2배로 한 스트레스 결과를 동시에 보고한다.

### 10.5 성능지표

- 총·연환산 수익률은 충분한 기간일 때만 보조 지표로 제공한다.
- 핵심 지표: 비용 후 기대값, 중앙값 거래손익, 승률, 평균이익/평균손실, profit factor, MDD, expected shortfall, turnover, exposure time, 평균·95백분위 슬리피지, 미체결률, 한쪽 다리 노출시간.
- bootstrap 신뢰구간과 날짜 단위 permutation 결과를 제공한다.
- 전체 이익 중 최고 하루·최고 3일의 기여도를 별도로 표시한다.

### 10.6 전략 승격 기준

모든 조건을 만족해야 `PASS`다.

- 테스트 또는 전진검증의 비용 후 거래당 기대값이 양수다.
- profit factor가 1.2 이상이다.
- 수수료·슬리피지 2배 스트레스에서도 누적손익이 0 이상이다.
- 단일 거래일이 총 양의 손익의 30%를 초과하지 않는다.
- 전략 배정자본 기준 MDD가 5% 이하다.
- 데이터 품질·미체결·한쪽 다리 위험이 사전 한도 안이다.

조건 일부가 표본 부족으로 판정 불가능하면 `HOLD`, 기준을 실패하면 `REJECT`다. 임계값을 낮춰 실패한 전략을 사후 구제하지 않는다.

## 11. 페이퍼트레이딩과 리스크

### 11.1 승격 전 최소 관측

- H2/H3는 최소 60거래일 전진 페이퍼트레이딩을 수행한다.
- 전략별 적격 신호가 최소 30건이어야 한다.
- 데이터 장애·휴장·고변동성일을 임의로 제외하지 않는다. 제외 사유는 사전에 정의된 품질·리스크 규칙이어야 한다.

### 11.2 기본 한도

| 한도 | 기본값 |
| --- | --- |
| 거래당 최대 위험 | 계좌 기준자산의 0.25% |
| 일일 최대 손실 | 1.0% |
| 누적 MDD 킬스위치 | 5.0% |
| 연장시간 주문 유형 | 지정가만 |

| 한쪽 다리 허용시간 | 체결모델에서 정의된 짧은 timeout, 초과 시 차단·청산 시뮬레이션 |

포지션 수량은 위험예산, stop distance, 최소 거래단위와 호가 유동성 중 가장 보수적인 제한으로 결정한다.

### 11.3 킬스위치

다음 조건에서 신규 페이퍼 주문을 차단한다.

- 핵심 공급자 연결 끊김, 시세 지연 또는 공급자 불일치

- 거래소 휴장·정지·VI·가격제한폭 또는 세션 상태 불명
- `ConversionStatus` 미확인·중단, 차입 불가 또는 비용 만료
- 한쪽 다리 미체결과 헤지 실패
- 일손실·MDD 한도 도달
- 예상비용이 예상총수익 이상
- 전략 설정 또는 데이터 스키마가 승인 버전과 다름

## 13. 비기능 요구사항

### 13.1 정확성과 재현성

- 동일 데이터·설정·코드에서는 동일 결과를 생성한다.
- 원시·정규화·특징·신호·주문·체결 계층을 분리한다.
- 데이터 수정은 덮어쓰기보다 새 버전으로 기록한다.
- 계산 단위, 반올림, 시간대 규칙을 테스트로 고정한다.

### 13.2 복원력

- API token 만료, rate limit, timeout과 WebSocket 재연결에 지수 백오프와 jitter를 적용한다.
- 재연결 후 누락 구간을 REST 또는 공식 일별 데이터로 보충하고 보충 여부를 표시한다.
- 한 공급자의 장애가 다른 수집기와 저장 계층을 중단시키지 않도록 격리한다.
- 데이터가 충분하지 않으면 추정값을 꾸며내지 않고 `BLOCK`한다.

### 13.3 보안

- 비밀값이 Git 상태, 리포트, 테스트 결과에 나타나지 않아야 한다.
- 토큰 권한과 만료를 최소화한다.
- 실주문 기능은 별도 승인·위협모델·운영 런북 없이는 빌드 대상이 아니다.

### 13.4 성능

- v1은 초저지연 HFT가 아니라 개인 인터넷 환경의 연구 시스템이다.
- 수신지연과 시계 오차를 측정해 모든 레코드에 남긴다.
- 지연 한도를 넘으면 더 빠른 추정으로 대체하지 않고 신호를 차단한다.

완료조건: 승격 기준으로 각 전략을 최종 판정한다.

### Phase 4 — 실거래 검토(별도 프로젝트)

v1에서 자동 진입하지 않는다. 실거래 검토에는 별도 사용자 승인, 브로커별 주문·대주 권한 확인, 세무·규제 검토, 독립 운영 런북, 장애복구·수동 킬스위치와 소액 제한 시험이 필요하다.

## 17. 제품 완료 기준

- 세 전략의 입력, 계산, 거래 가능 조건, 반증 조건이 코드·테스트로 옮길 수 있게 명시되어 있다.
- KRX/KIS/Toss/공식 공시/포털의 역할과 우선순위가 중복 없이 정의되어 있다.
- 모든 공개 타입의 필드, 단위, 시간과 품질 상태가 정의되어 있다.
- 비용·부분체결·차입·전환·휴장·DST·한쪽 다리 위험이 백테스트에서 누락되지 않는다.
- 짧은 표본을 CAGR로 과장하지 않고 PASS/HOLD/REJECT 기준을 적용한다.
- 실주문 경로가 비활성화되어 있고 비밀값이 저장소에 포함되지 않는다.
- `background.md`, `draft_prd.md` 원문은 변경되지 않는다.

## 20. 문서 관리

- `prd.md`는 제품 요구사항과 수용 기준의 단일 진실 공급원이다.
- `background.md`는 문제 제기의 원문, `draft_prd.md`는 초기 아이디어 기록으로 보존하며 사실 근거로 자동 승격하지 않는다.
- 시장 구조·규제·API 계약이 바뀌면 정보 기준일과 해당 요구사항을 함께 갱신한다.
- 전략 임계값이나 승격 기준 변경은 변경 이유와 이전 결과에 미치는 영향을 기록한다.
- 구현 단계에서 아키텍처, 데이터·백테스트, 테스트 계획은 별도 문서로 분리하되 이 PRD의 요구사항 ID를 역참조한다.
