## 결정 경계

이 `CONFIRMED`는 다음 질문만 해소한다.

1. 기준 거래일의 무료 KRX ETF/ETN 응답에서 국내 단일종목 H1 상품을 동적으로 찾는가.
2. 상품명과 기초지수명으로 목표배율·기초자산을 해석하고 실패 상품을 제외하는가.
3. ETF `NAV × LIST_SHRS`, ETN `PER1SECU_INDIC_VAL × LIST_SHRS`를 실제 AUM이 아닌
   `listed_notional_proxy`로 계산하는가.
4. `BAS_DD < signal_date`, 실제 `received_at_utc < as_of_time_utc`, 비어 있지 않은 원천
   record ID를 강제하는가.
5. 결과가 `daily-proxy`·승격 비대상으로 원래 H1과 분리되는가.

다음 항목은 의도적으로 G-04 필수조건에서 제외한다.

- PCF·구성종목과 복제방식
- 장중 iNAV/iIV와 NAV/IV의 실제 시·분 단위 `published_at`/`effective_at`
- 0이 아닌 공식 AUM/지표가치총액
- 15:10 장중 수익률·호가·예상체결·프로그램매매
- HKEX 및 선택된 국내 H1 universe 밖의 상품
- KIS/Toss 실시간 feed 구현

따라서 G-04 확인을 원래 H1의 페이퍼·실시간 실행 허가나 PASS 근거로 해석하면 안 된다.

- [x] KRX 실제 2026-07-16 응답에서 ETF 16개·ETN 2개, 총 18개 국내 단일종목
      레버리지 상품을 발견했고 삼성전자·SK하이닉스 각 9개로 분류했다.

      `BAS_DD`, ETF `NAV`/ETN `PER1SECU_INDIC_VAL`, `LIST_SHRS`를 보존하고 beta 해석
      실패를 exclusion으로 기록한다.
- [x] `application.h1_krx_daily_proxy`가 listed-notional, 일별 수익률, 20일 ADV와
      기존 theoretical exposure·close pressure 계약을 연결한다.
- [x] 동일 builder가 직전 기준일, as-of 수신시각, fund·market 원천 record ID를
      검증하며 결측·미래·동일일 입력을 거부한다.
- [x] `H1CloseRebalanceStrategy`가 promotion scope 혼합을 거부하고,
      `PromotionInput/Result`와 `ExperimentResult`가 proxy의 원 H1 PASS 승격을 차단한다.
- [x] fixture·단위·통합 테스트가 universe→proxy feature→strategy→기존 engine→promotion
      경로와 model/resolution/scope/lineage tag를 검증한다.
