# PRD 9.1 개정 초안 제안서 (PRD 9.1 Amendment Proposal)
> **상태:** 제안 상태 (PROPOSED) — 사용자 승인 대기 중  
> **기준일:** 2026-07-19  
> **대상 문서:** [`prd.md`](../../prd.md) 9.1장 (종가 리밸런싱 충격)  

---

## 1. 개정 필요성 및 배경

### 1.1 시간선 불일치 (Empty Intersection)
현재 `prd.md` 9.1장에 정의된 H1 전략의 시간선은 다음과 같습니다:
*   **기본 신호 스냅샷 시점:** 15:10:00 KST
*   **신규 페이퍼 주문의도 생성 마감 시점:** 15:19:30 KST

그러나 조사 결과([G-03-investigation.md](../decisions/gates/evidence/G-03-investigation.md))에 따르면, KRX의 실시간 종가 예상체결 가격 및 수량 데이터(`antc_cnpr`, `antc_vol`)가 생성 및 공표되는 실시간 예상체결 시간대(단일가 매매 세션)는 **15:20 ~ 15:30 KST**입니다.

따라서 현 규정 하에서는 아래와 같이 예상체결 정보의 가용 구간과 의사결정 허용 구간의 교집합이 존재하지 않습니다:
$$\text{종가경매 예상체결 가용 구간 (15:20} \sim \text{15:30)} \cap \text{의사결정 허용 구간 (} \le \text{15:19:30) } = \emptyset$$

이로 인해 원 15:10 H1 모델 및 KIS 실시간 추정 모델 모두 종가 경매의 실제 예상체결 정보(`H0STASP0`)를 활용할 수 없는 구조적 모순에 직면해 있으며, 이를 강제로 사용할 경우 미래 데이터를 15:10으로 소급하는 치명적인 **룩어헤드 편향(Lookahead Bias)**이 발생하게 됩니다.

---

## 2. 구체적 개정안 (H1-Extended / Close-Call 모델용)

실제 종가경매 단일가 시계열 데이터를 분석하고 활용하기 위해 새로운 모델 버전인 **`h1_kis_close_call_indicative_v1`**의 도입을 제안하며, 이에 맞춘 시간선 및 거래 규칙 개정을 제안합니다.

### 2.1 신규 시간선 정의
*   **신규 신호 스냅샷 시점:** **15:25:00 KST** (단일가 세션 개시 5분 후, 예상체결가의 노이즈가 완화되고 수렴하기 시작하는 시점)
*   **신규 페이퍼 주문의도 생성 마감 시점:** **15:28:00 KST** (KRX 주문접수 종료 2분 전으로 마감하여 네트워크 지연 및 버퍼 확보)

### 2.2 지연 및 신선도(Freshness) 규칙
*   **호가 신선도 임계값:** 스냅샷 시점 대비 데이터 수신 시각 차이(`decision_time - event_time`) **2초 이하** 강제.
*   **주문의도 취소 규칙:** 15:28:00 이전에 생성된 주문의도는 15:29:00 KST까지 단일가 예상체결가의 변동폭이 직전 대비 $\pm 2\%$를 초과할 경우 자동 취소(Cancel) 처리하여 이상 변동 리스크를 방지함.

### 2.3 체결 및 비용 모델 분리
*   **체결 모델:** 연속장 지정가 체결 모델에서 완전히 탈피하여, 단일가 매매의 다대다 매칭(Clearing) 메커니즘을 시뮬레이션함. 최종 15:30 확정 종가(Official Close Price)로만 체결되는 것을 원칙으로 함.
*   **시장 충격(Slippage) 모델:** 주문 수량이 당시 예상체결 잔량($V_e$)의 일정 비율(예: $10\%$)을 초과할 경우, 초과분에 대해 단일가 호가 스프레드 단계를 밀어내며 발생하는 슬리피지를 가산함.
*   **비용 모델:** 단일가 세션 진입에 따른 청산 비용 및 2배 비용 스트레스 조건을 별도 적용함.

## 3. 기존 연구 경로와의 관계

본 개정안은 기존 H1 연구 경로를 대체하지 않으며, 상호독립적인 별도 트랙으로 병렬 관리됩니다:

1.  **`h1_krx_daily_proxy_reduced_v1` (일별 프록시 경로):**
    *   일별 종가 데이터를 기반으로 장기 백테스트를 수행하는 경로로, 실시간 15:10/15:20 시간선 제약에 영향을 받지 않으며 기존 승격 범위(`h1-daily-proxy-research-only`)를 유지합니다.
2.  **`h1_kis_continuous_ofi_program_proxy_v1` (연속장 OFI 프록시 경로):**
    *   현행 `prd.md` 규칙(15:10 스냅샷, 15:19:30 마감)을 그대로 준수하되, 15:00~15:10 연속장 호가 흐름과 프로그램 매매 흐름을 대용치로 사용하는 모델로, 본 개정안과 섞이지 않고 독립적으로 '값싸게 기각' 여부를 검증합니다.
3.  **`h1_kis_close_call_indicative_v1` (제안 모델 경로):**
    *   본 개정안 승인 시에만 활성화되는 신규 경로로, 승격 범위는 **`h1-close-call-indicative-research-only`**로 분리하여 관리합니다.

## 4. 룩어헤드(Lookahead) 방지 원칙의 고수

*   신호 스냅샷 시점이 15:25:00로 이동하더라도, 백테스트 엔진 내의 **인과적 시간 경계 필터는 엄격히 유지**됩니다.
*   `event_time_utc <= 15:25:00 KST` 및 `received_time_utc <= 15:28:00 KST`를 충족하는 레코드만 피처 생성에 사용 가능하며, 15:25:01 이후에 발생한 모든 시세 및 15:30 확정 종가는 피처 히스토리에 절대로 포함될 수 없습니다.

## 5. `prd.md` 9.1장 문구 개정안 제안 (Diff)

사용자 승인 시 `prd.md` 9.1장에 적용될 정확한 변경 diff는 다음과 같습니다. 이 변경은 기존 일별 프록시 결과나 연속장 프록시 백테스트 결과에 하위 호환성 충돌을 일으키지 않습니다 (버전 명시적 분리).

```diff
- - 기본 신호 스냅샷: KRX 15:10:00 KST.
- - 신규 페이퍼 주문 의도 생성 마감: 15:19:30 KST.
+ - 기본 신호 스냅샷 (H1-Original 및 연속장 프록시): KRX 15:10:00 KST.
+ - 신규 페이퍼 주문 의도 생성 마감 (H1-Original 및 연속장 프록시): 15:19:30 KST.
+ - 신규 신호 스냅샷 (H1-Close-Call 단일가 모델): KRX 15:25:00 KST.
+ - 신규 페이퍼 주문 의도 생성 마감 (H1-Close-Call 단일가 모델): 15:28:00 KST.
```

## 6. 미결정 사항 및 잔여 리스크

1.  **호가 잔량 변동성 리스크 (Order Book Spoofing):**
    *   단일가 매매 시간(15:20~15:30) 동안 허수 주문 및 주문 취소가 빈번하게 발생하여, 15:25:00 기준의 예상체결 정보가 15:30 최종 체결 시점에 심각하게 왜곡될 리스크가 상존합니다.
2.  **호가 수집 레이턴시 리스크:**
    *   단일가 세션 개시 직후 발생하는 대규모 주문 폭주로 인해 KIS API 서버의 웹소켓 패킷 전송 지연이 발생할 수 있으며, 이로 인한 신호 생성 시점 지연(2초 초과로 인한 무신호 차단 비율 증가) 위험이 있습니다.
3.  **체결 실패 리스크:**
    *   15:28:00 주문 제출 후 종가 결정 직전 대규모 반대 주문 유입으로 인해 호가 제한폭에 걸리거나 미체결이 발생할 시의 리스크 대응 로직이 추가적으로 연구되어야 합니다.
